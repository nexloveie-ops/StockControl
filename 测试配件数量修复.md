# 快速测试：配件数量修复

## 🎯 测试目标
验证群组库存中配件产品的可调货数量是否正确显示

## 📋 准备
1. 清除浏览器缓存（Ctrl+F5）
2. 登录 MurrayRanelagh 账号

## 🧪 测试步骤

### 步骤 1: 验证页面版本
1. 打开浏览器控制台（F12）
2. 刷新页面
3. ✅ 应该看到：`✅ 页面版本: v2.1.4 - 修复群组库存配件数量显示`

### 步骤 2: 进入群组库存
1. 点击"群组库存"标签
2. 选择店铺：MurrayDundrum
3. 选择分类（包含 Phone Case 的分类）

### 步骤 3: 测试 Phone Case, iPhone 13 - Clear
1. 找到产品：Phone Case
2. 查看显示的总库存：______ 件（应该是 3）
3. 点击"🎨 选择型号和颜色"按钮
4. 在弹出的模态框中找到：iPhone 13 - Clear
5. 查看该变体的库存：______ 件（应该是 3）
6. 点击选择该变体

### 步骤 4: 验证可用数量
1. ✅ 应该弹出提示：`请输入调货数量（可用：3）`
2. 输入数量：2
3. 点击确定
4. ✅ 应该显示：`已添加 2 件 Phone Case 到调货清单`

### 步骤 5: 验证购物车
1. 查看右侧购物车
2. ✅ 应该显示：
   - 产品名称：Phone Case
   - 数量：2
   - 可以使用 +/- 按钮调整数量

### 步骤 6: 测试库存限制
1. 再次选择相同的变体（iPhone 13 - Clear）
2. 输入数量：2
3. ✅ 应该提示：`库存不足，最多可调货 3 件`
4. 输入数量：1
5. ✅ 应该成功添加，购物车中数量变为 3

### 步骤 7: 测试超出库存
1. 再次选择相同的变体
2. 输入数量：1
3. ✅ 应该提示：`库存不足，最多可调货 3 件`

## ✅ 测试通过标准
- [ ] 页面版本正确（v2.1.4）

- [ ] 可用数量提示正确（可用：3）
- [ ] 可以添加正确的数量到购物车
- [ ] 库存限制正常工作
- [ ] 购物车显示正确
