## 🔐 安全建议

### 生产环境使用

⚠️ **重要**: 这些是测试账号，密码非常简单！

在生产环境中，请务必：

1. **修改默认密码**
   ```
   - 使用强密码（至少8位，包含大小写字母、数字、特殊字符）
   - 定期更换密码
   ```

2. **创建专用管理员账号**
   ```
   - 不要使用默认的 admin 账号
   - 为每个管理员创建独立账号
   - 使用真实的邮箱地址
   ```

3. **限制管理员数量**
   ```
   - 只给必要的人员管理员权限
   - 使用仓库管理员角色代替管理员（如果不需要用户管理权限）
   ```

4. **启用日志记录**
   ```
   - 记录所有登录尝试
   - 记录所有管理操作
   - 定期审查日志
   ```

## ❓ 常见问题

### Q1: 忘记密码怎么办？

**A**: 可以通过以下方式重置：
1. 使用管理员账号登录，修改该用户的密码
2. 运行 `create-initial-users.js` 脚本（会跳过已存在的用户）
3. 直接在数据库中修改（需要bcrypt加密）

### Q2: 如何删除初始账号？

**A**: 
1. 使用管理员账号登录
2. 进入 "👥 用户管理"
3. 找到要删除的用户
4. 点击 "停用"（软删除）

⚠️ 注意: 不建议删除 admin 账号，除非有其他管理员账号。

### Q3: 可以修改用户名吗？

**A**: 目前不支持修改用户名。如果需要更改，请：
1. 创建新用户（使用新用户名）
2. 停用旧用户

### Q4: 登录后跳转到错误的页面？

**A**: 检查以下内容：
1. 用户角色是否正确
2. `auth-guard.js` 中的角色映射是否正确
3. 清除浏览器缓存和localStorage
